Square-One Simulator

Intro

This project is a simulator for my favorite twisty-puzzle: the Square-One. You can look at the source code here: link.


Before I go on to the project, I think it would be a good idea to show you how a Square-One works. If you're familiar with Squan, just scroll down a bit.


The Square-One in its solved state looks like a cube. But do you see how the pieces look different? Like a 3x3, there are two types of pieces that move around: edges and pieces. Unlike a 3x3, edges have an internal angle of 30 degrees and corners have an angle of 60 degrees (which makes corners larger). The top and bottom layers can rotate and the "slice" can too, but only in increments of 180 degrees. This slice is at a 30 degree angle too.



Modeling

It took me a while to figure out how to express a Square-One's state in terms of code, but I figured it out eventually. I modeled the cube as 2 arrays, each representing a layer (top or bottom), with 12 numbers in them. Each edge has one number and each corner two numbers. Finally, the "equator" (the bar in the middle that flips around) is also either 'solved' or 'not solved', which is easily represented as a boolean.

To the right is a diagram of a Square-One with the pieces in the solved state. The first array ("up") would contain elements 1 through 12 and the second array ("down") would contain elements 13 through 24. Notice that because each corner has two numbers, those number MUST stick be in the same array and next to each other. For example, having "8" in one array and "9" in the other would be an invalid state because a corner cannot separate across layers. However, I also had to make sure that the arrays allowed looping so that corners could theoretically be over a slice/edge.



The Program + Features

I wrote this program in Java with the Processing library for a GUI.



Features:

  • Accurately captures movement of pieces and prevents user from entering an invalid state.
  • Command-line input that allows users to type in algorithms and other commands.
  • Displays last algorithm or command executed in the top center. Also outputs error messages.
  • Multiple commands: reset (for auto-solving the puzzle), invert (inverting algorithms), stepthrough (for going through an algorithm one step at a time), and commands to toggle cosmetic options.
  • Pasting capabilities for the command line.
  • Saves state of puzzle and cosmetic options when closing the applet.
  • Cosmetic Options:
    • Karnaukh Notation: This is a more streamlined version of the normal Square-One notation.
    • Color Scheme: Some people prefer black tiles instead of yellow, so I added an option to toggle tile color.
    • Cap Scheme: The side tiles are switchable, which switches the top and bottom colors in the solved state.
    • Rendering: 2D and 3D rendering options available (but 3D is cooler).
